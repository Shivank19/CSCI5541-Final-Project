# Tier 3: LLM Prompting and Evaluation Pipeline

## Overview

This directory contains the Tier 3 Large Language Model (LLM) zero-shot and few-shot prompting pipeline for detecting linguistic precursors to financial restatements (SEC Form 8-K Item 4.02).

The pipeline evaluates transcripts under two explicit structural conditions, **Scripted Prepared Remarks** and **Unscripted Analyst Q&A**, testing the hypotheses that cognitive load and deception manifest differently across these segments (H1, H2, and H4). We compare general-purpose LLMs (Llama 3) against finance-pretrained models (FinGPT) to assess whether domain-adaptive pretraining offers an advantage over raw reasoning scale (H3).

## Hardware and Environment Requirements

This pipeline requires running models ranging from 8 billion to 70 billion parameters. Due to strict VRAM constraints, different models require different hardware tiers.

### Recommended Hardware

| Model | Hardware Requirements | VRAM / Memory |
|---|---|---|
| Llama 3 (8B) | Mac M1/M2/M3 or PC with dedicated GPU | 16 GB+ unified memory / 8 GB+ VRAM |
| FinGPT (Llama-2-13B) | Google Colab Pro (A100 GPU) | Approximately 26 GB VRAM (16-bit precision) |
| Llama 3 (70B) | Google Colab Pro (H100 GPU) | Approximately 40 GB VRAM (4-bit quantized) |

**Note:** FinGPT and Llama 3 70B cannot be run simultaneously on a single Colab A100 instance. They will collide and cause an out-of-memory (OOM) crash. They must be executed in isolated sessions.

### Software Dependencies

- Ollama: Must be installed and running as a background service.
- Python libraries: `torch`, `transformers`, `accelerate`, `peft`, `sentencepiece`, `pandas`, `scikit-learn`, `tqdm`, `ollama`

## Results, Interpretations, and Hypothesis Testing

The outputs of this pipeline directly address the core hypotheses outlined in the project proposal.

### 1. FinGPT Catastrophic Degradation (Addressing H3)

The project explored whether domain-adaptive pretraining confers an advantage over general-purpose LLMs without explicit fine-tuning.

**Finding:** Narrow domain pretraining degraded complex reasoning. When subjected to the hypothesis-aware zero-shot prompts requiring complex deception detection, FinGPT (`fingpt-sentiment_llama2-13b_lora`) suffered catastrophic architectural degradation.

**Reasoning:** Because the LoRA was narrowly fine-tuned strictly for ternary sentiment classification on datasets such as Financial PhraseBank, it failed to generalize, resulting in a parse rate of `0.03` and severe token hallucination.

**Conclusion:** General-purpose LLMs (Llama 3) proved drastically more robust for detecting complex linguistic deception.

### 2. General-Purpose Model Scaling (8B vs. 70B)

**Llama 3 (8B):** Successfully navigated the complex JSON schema (`Parse Rate = 1.0`). However, AUROC scores hovered near random chance (`0.50 - 0.54`), showing it lacks the reasoning capacity to uncover subtle epistemic hedging.

**Llama 3 (70B):** Scaling to the 70B model yielded calibrated decision boundaries and raised the AUROC to `0.611`, demonstrating that sufficient parameter count is required to extract subtle signals from complex financial text.

### 3. The Overcompensation Hypothesis (H4) Validation

The dynamic validation loop provided structural evidence for H4 (Overcompensation).

**Validation:** When evaluating the `scripted_text` condition, the model achieved higher PR-AUC using the `section_aware` prompt, a prompt specifically instructing the model to look for "unusually upbeat or defensive language that may overcompensate for hidden problems."

**Conclusion:** This validates the theory that executives use prepared remarks as a linguistic smokescreen, structurally independent of the evasion found in live Q&A.

### 4. H2 vs. H4: Overcompensation Dominates

A central methodological contribution of this project was splitting calls into Scripted versus Unscripted sections. The initial hypothesis (H2) was that the cognitive load of unscripted Q&A would leak the most detectable deception signals.

However, the 70B results showed the opposite: the model achieved a higher AUROC on the Scripted Prepared Remarks (`0.611`) than on the Unscripted Q&A (`0.580`). Furthermore, the pipeline selected the `section_aware` prompt for the scripted condition, achieving peak performance when explicitly instructed to flag defensive overcompensation.

**Summary Conclusion:** The highly manufactured, overly optimistic linguistic façade executives deploy in prepared remarks is a more statistically reliable precursor to an SEC 8-K Item 4.02 restatement than their unscripted evasion.
